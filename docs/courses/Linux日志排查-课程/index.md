# Linux 日志排查 — 渐进式学习课程

## 核心本质

日志是系统的**黑匣子**——它忠实记录了"发生过什么"。而日志排查的本质，是**在海量噪声中快速定位关键信息的能力**。

Linux 的设计哲学是"一切皆文件"，日志也不例外。Linux 提供了一套强大的文本处理工具链，每个工具只做一件事，但通过管道组合可以完成极其复杂的查询。掌握日志排查，就是掌握**"用正确的工具向正确的数据提出正确的问题"**这门手艺。

## 实战项目

**项目：排查一个线上 Go Web 服务的故障**

模拟一个真实的 Go HTTP 服务（订单系统）运行在 CentOS 上，它会产生各种典型问题：接口超时、panic 崩溃、内存泄漏、错误率飙升。每一章学到的命令，都会立刻用于排查这个服务的某个具体故障。选它是因为这就是你日常工作中会遇到的场景。

## 课程大纲

### 第1章：日志的世界观 — Linux 日志体系全景
认识 Linux 上日志住在哪里、分哪些种类（系统日志 vs 应用日志 vs 内核日志），建立"出了问题该去哪找"的直觉。产出：画出你服务器上的日志地图。

### 第2章：看日志的基本功 — cat, less, head, tail, grep
掌握最高频的 5 个命令，解决"怎么打开日志、怎么翻页、怎么搜关键字"这三个最基础的问题。产出：用 grep 从项目日志中定位一次 500 错误。

### 第3章：文本处理三剑客 — awk, sed 与统计分析
当 grep 不够用时，用 awk 按列提取、用 sed 做替换清洗、用 sort/uniq/wc 做统计。产出：统计出某小时内各接口的错误次数排行。

### 第4章：实时追踪与系统日志 — tail -f, journalctl, dmesg
从"事后翻日志"进化到"实时盯日志"，同时学会读懂系统级日志（OOM Kill、磁盘满、服务崩溃）。产出：实时监控 Go 服务并捕获一次 panic。

### 第5章：Go 项目的日志规范 — 结构化日志与 slog/zap
从排查者的视角反推：什么样的日志才好查？学习结构化日志、日志级别设计、Go 常用日志库的最佳实践。产出：为项目接入 slog，输出结构化 JSON 日志。

### 第6章：管道组合术 — 复杂场景的排查链路
将前面所有命令通过管道 `|` 串联，配合 xargs、tee、进程替换等技巧，构建一条完整的排查链。产出：一行命令定位"最近1小时响应最慢的10个请求及其上下文"。

### 第7章：综合实战 — 从报警到定位的完整排查流程
整合全部知识，模拟一次真实的线上故障：收到报警 → 判断方向 → 定位根因 → 验证修复。复盘整个课程的知识脉络。产出：完成一次端到端的故障排查报告。
