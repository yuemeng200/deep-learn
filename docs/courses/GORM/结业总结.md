# 结业总结

## 1. 核心理念提炼

**理念一：GORM 是声明式的，不是命令式的**
你用 struct 和 tag 声明"数据长什么样"，用链式调用声明"我要什么数据"。GORM 负责翻译成 SQL。你不需要想 SQL，但你需要理解 GORM 会生成什么 SQL。

**理念二：零值问题是 Go 的，不是 GORM 的**
这不是 GORM 设计不好，而是 Go 没有可选类型。接受它，用 `map` 和 `Select` 对付它。

**理念三：关联是 GORM 最强也最容易出问题的部分**
预加载、Association 方法、外键约定——这些用对了很省事，用错了生成的 SQL 会让你困惑。遇到关联问题，`Debug()` 看实际 SQL 是最快的排查方式。

## 2. 学员评估

学习风格偏实用，能快速判断哪些概念和已有经验重合、哪些需要深究。已经具备用 GORM 构建完整数据层的能力，可以在实际项目中上手。遇到复杂场景（多态关联、自定义类型、批量操作）时查阅文档即可。

## 3. 进阶方向

- **多态关联与自定义中间表**：Comment 可以属于 Task 也可以属于 Project——用多态关联实现
- **性能优化**：批量操作（CreateInBatches）、索引策略、查询优化
- **数据库迁移工具**：golang-migrate / Atlas，替代 AutoMigrate 管理生产迁移
- **GORM Gen**：GORM 官方的代码生成工具，把查询从运行时反射变成编译时类型安全
