# 第3章：关联关系

> 前两章处理的是单表。真实业务中数据之间是有关系的——GORM 用 struct 字段 + tag 来表达这些关系，这正是"用 struct 表达关系模型"的核心延展。

## 3.1 四种关联一览

在 TaskFlow 中，我们引入更多模型来覆盖全部关联类型：

```
User ──HasMany──→ Task          （一个用户创建多个任务）
Task ──BelongsTo─→ User         （任务属于某个用户）
User ──HasOne───→ UserProfile   （一个用户一个档案）
Task ←──Many2Many──→ Tag        （任务和标签多对多）
```

先定义完整的模型：

```go
// models/user.go
type User struct {
    gorm.Model
    Name      string
    Email     string `gorm:"uniqueIndex"`
    Profile   UserProfile      // HasOne
    Tasks     []Task           // HasMany
}
```

```go
// models/user_profile.go
type UserProfile struct {
    gorm.Model
    UserID   uint   // BelongsTo 的外键（默认约定：拥有者模型名 + ID）
    Bio      string
    Avatar   string
}
```

```go
// models/task.go
type Task struct {
    gorm.Model
    Title       string
    Status      string `gorm:"default:'todo'"`
    CreatorID   uint           // BelongsTo 外键
    Creator     User           // BelongsTo
    Tags        []Tag `gorm:"many2many:task_tags;"` // Many2Many
}
```

```go
// models/tag.go
type Tag struct {
    gorm.Model
    Name  string `gorm:"unique"`
    Tasks []Task `gorm:"many2many:task_tags;"`
}
```

## 3.2 外键约定

GORM 的关联靠**外键约定**工作：

| 关联类型 | 外键位置 | 默认外键名 |
|---------|---------|-----------|
| BelongsTo | 子表 | `引用模型名 + ID`（如 `UserID`） |
| HasOne | 子表 | `拥有者模型名 + ID` |
| HasMany | 子表 | `拥有者模型名 + ID` |
| Many2Many | 中间表 | 两个模型各自的主键 |

如果外键名不遵循约定，用 tag 指定：

```go
type Task struct {
    gorm.Model
    AssigneeID uint   // 自定义外键名
    Assignee   User   `gorm:"foreignKey:AssigneeID"`
}
```

## 3.3 关联操作

**创建时关联**

```go
// 创建用户并同时创建其档案（HasOne）
DB.Create(&User{
    Name:  "张三",
    Email: "zhangsan@test.com",
    Profile: UserProfile{Bio: "Go 开发者"},
})
// INSERT INTO users ...
// INSERT INTO user_profiles (user_id, bio, ...) ...

// 创建任务并关联标签（Many2Many）
tag1 := Tag{Name: "紧急"}
tag2 := Tag{Name: "前端"}
DB.Create(&Task{
    Title:   "修复首页 bug",
    Creator: user,
    Tags:    []Tag{tag1, tag2},
})
// INSERT INTO tasks ...
// INSERT INTO task_tags (task_id, tag_id) VALUES (1,1), (1,2)
```

**查询关联**

```go
// 查用户时带上档案
var user User
DB.Preload("Profile").First(&user, 1)
fmt.Println(user.Profile.Bio)

// 查任务时带上创建者和标签
var task Task
DB.Preload("Creator").Preload("Tags").First(&task, 1)
```

**单独操作关联**

```go
// 给已有任务添加标签
DB.Model(&task).Association("Tags").Append([]Tag{{Name: "新标签"}})

// 替换所有标签
DB.Model(&task).Association("Tags").Replace([]Tag{tag1, tag3})

// 删除关联（不删记录，只删中间表）
DB.Model(&task).Association("Tags").Delete(tag1)

// 清空关联
DB.Model(&task).Association("Tags").Clear()

// 统计关联数量
count := DB.Model(&task).Association("Tags").Count()
```

**查询关联记录列表**

```go
// 方法一：用 Association
var tags []Tag
DB.Model(&task).Association("Tags").Find(&tags)

// 方法二：直接通过 Preload + 条件
var tasks []Task
DB.Preload("Tags", "name LIKE ?", "%紧急%").Find(&tasks)
```

## 3.4 级联删除

默认情况下，删除记录**不会**删除关联记录。如果需要级联，用 `Select` 在删除时指定要一起删的关联：

```go
// 删除用户时同时删除其档案和任务
DB.Select("Profile", "Tasks").Delete(&user)

// 删除所有关联
DB.Select(clause.Associations).Delete(&user)
```

## 3.5 完整模型汇总

运行 `AutoMigrate` 创建所有表：

```go
database.DB.AutoMigrate(
    &models.User{},
    &models.UserProfile{},
    &models.Task{},
    &models.Tag{},
)
```

项目结构现在是：

```
taskflow/
├── main.go
├── go.mod
├── database/
│   └── db.go
└── models/
    ├── user.go
    ├── user_profile.go
    ├── task.go
    └── tag.go
```

## 章末思考题

1. **Task 有 `CreatorID uint` 和 `Creator User` 两个字段。** GORM 怎么知道 `Creator` 是关联而不是普通字段？如果你把 `CreatorID` 删掉，只留 `Creator User`，GORM 还能正确创建外键列吗？

2. **Many2Many 的中间表 `task_tags` 是自动创建的。** 如果你想在中间表上额外加字段（比如"谁打的标签" `user_id`），直接给中间表加列可以吗？想想 GORM 的 `AutoMigrate` 会怎么处理。

3. **`Association("Tags").Append()` 和直接更新 `task.Tags` 然后 `DB.Save(&task)` 都能给任务加标签。** 这两种方式生成的 SQL 有什么不同？哪种更推荐？
