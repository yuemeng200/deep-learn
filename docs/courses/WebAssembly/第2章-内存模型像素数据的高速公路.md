# 第2章：内存模型——像素数据的高速公路

上一章我们理解了 Wasm 存在的意义，也跑通了第一个模块。但留下了一个关键问题：`wasm-bindgen` 自动转换数据很方便，却会带来复制开销。处理 200 万个像素时，这种开销不可接受。本章我们就来解决它——让 JS 和 Wasm **共享同一块内存**，直接在上面读写像素数据。

---

## 2.1 Wasm 的线性内存：一张巨大的白纸

Wasm 的内存模型极其简单，简单到可以用一句话概括：

> **一段连续的字节数组，下标从 0 开始，所有人都能直接读写。**

这就是"线性内存"（Linear Memory）。没有对象、没有引用、没有 GC——只有一堆排好队的字节。

```
地址:  0    1    2    3    4    5    6    7    8    ...
      ┌────┬────┬────┬────┬────┬────┬────┬────┬────┐
      │ R  │ G  │ B  │ A  │ R  │ G  │ B  │ A  │ ...│  ← 像素数据
      └────┴────┴────┴────┴────┴────┴────┴────┴────┘
       像素0 (0,0)              像素1 (1,0)
```

关键特性：

- **JS 可以拿到这段内存的引用**，以 `Uint8Array` 或 `Uint32Array` 的形式直接读写
- **Rust 可以拿到同一段内存的引用**，以 `&mut [u8]` 切片的形式直接读写
- **没有复制**——两边看到的是同一块内存，改了一边另一边立刻可见

这意味着什么？**JS 把像素数据写进这块内存，调用 Wasm 函数处理，Wasm 直接在原地修改，JS 读出来的就是处理后的结果。零拷贝。**

---

## 2.2 两种共享内存的方式

Wasm 和 JS 共享内存有两种途径：

**方式一：Wasm 导出自己的内存**

Rust 侧声明一块内存，导出给 JS 使用。这是最常见的方式。

```rust
// Rust 侧：导出 Wasm 内存
use wasm_bindgen::prelude::*;

#[wasm_bindgen]
pub fn get_memory() -> JsValue {
    // 返回 Wasm 线性内存的引用
    wasm_bindgen::memory()
}
```

JS 侧拿到后，就是一段普通的 `ArrayBuffer`：

```javascript
const memory = engine.get_memory();
const pixels = new Uint8Array(memory.buffer);
// 现在 JS 和 Rust 共享 pixels 这段内存
```

**方式二：JS 导入自己的内存给 Wasm**

JS 创建 `WebAssembly.Memory`，传给 Wasm 使用。这种方式 JS 侧控制更灵活，适合需要精确管理内存的场景。

```javascript
// JS 侧创建 1 页内存（每页 64KB）
const memory = new WebAssembly.Memory({ initial: 256 }); // 256 页 = 16MB

// 把 memory 传给 Wasm 模块（通过初始化选项）
```

> 对于视频编辑器，我们用**方式一**——让 Rust 管理内存，JS 只是"借来用"。原因：Rust 的内存分配器会帮我们处理分配和释放，JS 侧不用操心。

## 2.3 实战：实现第一个图像滤镜

现在我们来实现真正有用的东西：用 Wasm 处理图像像素。

### 步骤一：扩展 Rust 引擎

在 `src/lib.rs` 中添加图像处理函数：

```rust
use wasm_bindgen::prelude::*;

/// 灰度化：将 RGBA 像素转为灰度
#[wasm_bindgen]
pub fn grayscale(data: &mut [u8]) {
    // data 是从 JS 传来的共享内存视图
    // 每 4 字节一组：R, G, B, A
    for chunk in data.chunks_exact_mut(4) {
        let r = chunk[0] as f32;
        let g = chunk[1] as f32;
        let b = chunk[2] as f32;

        // 人眼对绿色最敏感，所以绿色权重最高
        let gray = (0.299 * r + 0.587 * g + 0.114 * b) as u8;

        chunk[0] = gray; // R
        chunk[1] = gray; // G
        chunk[2] = gray; // B
        // chunk[3] 是 Alpha，保持不变
    }
}

/// 调节亮度
/// factor > 0 变亮, factor < 0 变暗, factor 范围建议 [-128, 128]
#[wasm_bindgen]
pub fn adjust_brightness(data: &mut [u8], factor: i16) {
    for chunk in data.chunks_exact_mut(4) {
        for i in 0..3 { // 只改 R, G, B，不动 A
            let val = chunk[i] as i16 + factor;
            chunk[i] = val.clamp(0, 255) as u8;
        }
    }
}

/// 调节对比度
/// factor = 0 不变, factor > 0 增加对比度, factor < 0 降低
#[wasm_bindgen]
pub fn adjust_contrast(data: &mut [u8], factor: i16) {
    let factor_f = factor as f32 / 100.0;
    for chunk in data.chunks_exact_mut(4) {
        for i in 0..3 {
            let val = chunk[i] as f32;
            // 以 128 为中心拉伸或压缩
            let adjusted = (val - 128.0) * factor_f + 128.0;
            chunk[i] = adjusted.clamp(0.0, 255.0) as u8;
        }
    }
}
```

注意这里的参数类型——`&mut [u8]` 是 Rust 的可变切片。`wasm-bindgen` 会把它映射为 JS 的 `Uint8Array`，并且**直接指向 Wasm 的线性内存**，不做任何复制。

### 步骤三：前端页面——实时滤镜

更新 `index.html`，加入图片加载和滤镜控制：

```html
<!DOCTYPE html>
<html lang="zh">
<head>
    <meta charset="UTF-8">
    <title>Wasm 图像滤镜</title>
    <style>
        body { font-family: sans-serif; max-width: 800px; margin: 2rem auto; }
        canvas { border: 1px solid #ccc; max-width: 100%; }
        .controls { margin: 1rem 0; }
        .controls label { display: inline-block; width: 80px; }
        .controls input { width: 200px; }
        button { padding: 0.5rem 1rem; margin: 0.25rem; cursor: pointer; }
    </style>
</head>
<body>
    <h1>Wasm 图像滤镜</h1>

    <div class="controls">
        <input type="file" id="fileInput" accept="image/*">
    </div>
    <div class="controls">
        <label>亮度</label>
        <input type="range" id="brightness" min="-128" max="128" value="0">
        <span id="brightnessVal">0</span>
    </div>
    <div class="controls">
        <label>对比度</label>
        <input type="range" id="contrast" min="-100" max="300" value="100">
        <span id="contrastVal">100</span>
    </div>
    <div class="controls">
        <button id="grayscaleBtn">灰度化</button>
        <button id="resetBtn">重置</button>
    </div>

    <canvas id="canvas"></canvas>

    <script type="module">
        import init, {
            grayscale,
            adjust_brightness,
            adjust_contrast
        } from './pkg/video_editor_engine.js';

        const canvas = document.getElementById('canvas');
        const ctx = canvas.getContext('2d');
        let originalData = null; // 保存原始像素数据用于重置

        async function main() {
            await init();
            setupUI();
        }

        function setupUI() {
            const fileInput = document.getElementById('fileInput');
            const brightnessSlider = document.getElementById('brightness');
            const contrastSlider = document.getElementById('contrast');

            // 加载图片
            fileInput.addEventListener('change', (e) => {
                const file = e.target.files[0];
                if (!file) return;
                const img = new Image();
                img.onload = () => {
                    canvas.width = img.width;
                    canvas.height = img.height;
                    ctx.drawImage(img, 0, 0);
                    // 保存原始数据
                    originalData = ctx.getImageData(0, 0, img.width, img.height);
                };
                img.src = URL.createObjectURL(file);
            });

            // 亮度滑块
            brightnessSlider.addEventListener('input', (e) => {
                document.getElementById('brightnessVal').textContent = e.target.value;
                applyFilters();
            });

            // 对比度滑块
            contrastSlider.addEventListener('input', (e) => {
                document.getElementById('contrastVal').textContent = e.target.value;
                applyFilters();
            });

            // 灰度按钮
            document.getElementById('grayscaleBtn').addEventListener('click', () => {
                const imageData = ctx.getImageData(0, 0, canvas.width, canvas.height);
                // 关键：直接把像素数据传给 Wasm，零拷贝！
                grayscale(imageData.data);
                ctx.putImageData(imageData, 0, 0);
            });

            // 重置按钮
            document.getElementById('resetBtn').addEventListener('click', () => {
                if (originalData) {
                    ctx.putImageData(originalData, 0, 0);
                    brightnessSlider.value = 0;
                    contrastSlider.value = 100;
                    document.getElementById('brightnessVal').textContent = '0';
                    document.getElementById('contrastVal').textContent = '100';
                }
            });
        }

        function applyFilters() {
            if (!originalData) return;
            const brightness = parseInt(document.getElementById('brightness').value);
            const contrast = parseInt(document.getElementById('contrast').value);

            // 每次从原始数据开始，避免滤镜叠加
            const imageData = new ImageData(
                new Uint8ClampedArray(originalData.data),
                originalData.width,
                originalData.height
            );

            // 先亮度，再对比度
            if (brightness !== 0) adjust_brightness(imageData.data, brightness);
            if (contrast !== 100) adjust_contrast(imageData.data, contrast);

            ctx.putImageData(imageData, 0, 0);
        }

        main();
    </script>
</body>
</html>
```

## 2.4 关键解析：`imageData.data` 和 `&mut [u8]` 是怎么对接的

调用 `grayscale(imageData.data)` 时发生了什么？

```
JS 侧                              Wasm 侧

imageData.data (Uint8ClampedArray)
        ↓ wasm-bindgen 自动转换
    Uint8Array (指向 Wasm 线性内存)
        ↓ 传给 Rust 函数
    &mut [u8] (Rust 切片，指向同一段内存)
        ↓ Rust 在原地修改
    字节值被改写
        ↓ 返回 JS
    Uint8ClampedArray 中值已更新
        ↓ putImageData
    Canvas 渲染新像素
```

注意 `applyFilters()` 中有一行关键代码：

```javascript
const imageData = new ImageData(
    new Uint8ClampedArray(originalData.data),  // 复制原始数据
    ...
);
```

这里故意做了**一次复制**——因为我们需要每次从原始图片重新开始，而不是在已处理的图上叠加。但在 Wasm 处理环节本身，`adjust_brightness(imageData.data, brightness)` 是零拷贝的。

> 核心原则：**数据搬运的决策权在你手里**。Wasm 不会强迫你复制，JS 也不会。你决定什么时候复制、什么时候共享。

---

## 2.5 Rust 侧的 `&mut [u8]`：零拷贝的秘密

为什么 `&mut [u8]` 能做到零拷贝？因为 `wasm-bindgen` 知道 `Uint8Array` 内部就是一段连续字节，而 Wasm 线性内存也是一段连续字节。它只需要把 `Uint8Array` 的指针和长度传给 Rust，Rust 就直接把它们当作切片来用——**两边看到的是同一段内存**。

但要小心一个陷阱：

```javascript
// 危险！如果 Wasm 内存增长（比如 Rust 分配了新内存），旧的视图会失效
const view = new Uint8Array(wasm_memory.buffer);
// ... 之后 Wasm 内部触发了 memory.grow() ...
view[0]; // 💥 TypedArray is detached!
```

不过在视频处理场景中，我们不会动态增长内存（帧大小是固定的），所以这个问题不会出现。如果将来遇到动态内存的场景，只需在每次操作前重新创建视图即可。

## 章末思考题

**题目一**：灰度化公式使用了不同的权重（R: 0.299, G: 0.587, B: 0.114），而不是简单的 `(R+G+B)/3`。为什么？如果用简单平均，会出现什么视觉问题？

**参考答案**：

人眼对不同颜色的敏感度不同——对绿色最敏感，红色次之，蓝色最弱。简单平均会让图像看起来"偏暗"，因为蓝色和红色被赋予了过高的权重，而人眼最敏感的绿色反而被拉低了。

BT.601 标准（0.299R + 0.587G + 0.114B）模拟了人眼的亮度感知曲线。这个原理在视频编辑中处处体现——比如色度子采样（4:2:0）之所以能大幅压缩而人眼几乎看不出区别，正是因为丢弃了人眼不敏感的色度信息，保留亮度信息。

---

**题目二**：当前 `applyFilters()` 每次触发时都从 `originalData` 复制一份数据再处理。如果图片很大（比如 4K 分辨率），这个复制开销有多大？有没有办法避免？

**参考答案**：

4K 分辨率（3840×2160）的 RGBA 数据约 33 MB。每次拖动滑块都复制一次，60fps 滑动时约 2 GB/秒的纯内存拷贝。虽然比 Wasm 处理快（纯 memcpy），但仍然是浪费。

优化思路：**在 Wasm 侧保存原始数据**。首次加载图片时，把像素数据复制一份存入 Wasm 线性内存。每次应用滤镜时，Rust 从保存的原始数据开始处理，直接输出到目标缓冲区——这样 JS 侧就不需要做任何复制了。这正好是下一章"管线"架构的思路。

---

**题目三**：`adjust_brightness` 和 `adjust_contrast` 是两个独立的函数，每次只做一件事。但实际视频编辑器中，用户可能同时调整亮度、对比度、饱和度等多个参数。把每个参数做成独立函数、每次串行调用，和把所有参数合并到一个函数中一次性处理，这两种方案在性能上有什么区别？

**参考答案**：

**内存访问模式是关键区别。**

串行调用：遍历所有像素 3 次（亮度、对比度、饱和度各一次），每次都要从内存读 4 字节、写 4 字节。总计 3 次完整遍历。

合并处理：只遍历一次，对每个像素依次应用所有变换。总计 1 次完整遍历。

差异在于**缓存命中率**。一帧 1080p 约 8 MB，而 CPU L2 缓存通常 256KB-1MB，数据不能全部放入缓存。每次遍历都会导致缓存未命中（cache miss）。合并处理把多次遍历压缩为一次，大幅减少缓存未命中次数，性能可以提升 2-3 倍。

这也是为什么真正的视频处理引擎会把滤镜做成**管线（pipeline）**——下一章的主题。
