# WebAssembly 渐进式学习课程

## 核心本质

WebAssembly 解决的是一个根本矛盾：**浏览器的计算能力远远不够**。JavaScript 是动态类型、JIT 编译的语言，它擅长处理 UI 交互和业务逻辑，但在面对视频编解码、图像处理、物理模拟这类"搬砖式"的密集计算时，先天不足。WebAssembly 提供了一套接近硬件的低级二进制指令格式，让浏览器能以接近原生的速度执行计算密集型任务。

它的设计哲学是**协作而非替代**——JavaScript 负责调度和 UI，Wasm 负责重活，两者通过共享内存和函数调用无缝配合。这不是一门新语言要颠覆一切，而是一个精准的性能补丁，补在 Web 平台最薄弱的环节上。

## 实战项目

**Web 视频编辑器核心引擎**

从零构建一个能在浏览器中实时处理视频的引擎，涵盖图像滤镜、实时特效、多轨合成、视频导出等完整链路。选择这个项目是因为：视频处理是 Wasm 最典型的应用场景之一，每一帧都需要对大量像素做密集运算，性能差距在 JS 和 Wasm 之间会被放大到肉眼可见的程度。项目会从单张图片处理开始，逐步扩展到视频帧处理、实时特效、最终完成一个可导出的编辑器原型。

## 课程大纲

### 第1章：浏览器为什么需要 WebAssembly

理解 Wasm 诞生的根本原因，建立"JS 管调度，Wasm 管计算"的心智模型。搭建 Rust + wasm-pack 开发环境，完成第一个 Wasm 模块的编译和调用。初始化视频编辑器项目骨架。产出：一个可运行的 Wasm 函数，被 JS 调用并返回结果。

### 第2章：内存模型——像素数据的高速公路

深入理解 Wasm 的线性内存模型，掌握 JS 与 Wasm 之间如何高效传递大量数据。以像素操作为切入点，实现 ImageData 的跨边界读写。产出：一套可对图片施加灰度、亮度、对比度滤镜的 Wasm 图像处理模块。

### 第3章：Rust 与 Wasm 的深度协作

掌握 wasm-bindgen 的高级用法——自动类型转换、传递复杂数据结构、在 Rust 中调用 JS 函数。让两个世界的交互从"搬字节"升级为"调对象"。产出：一个支持滤镜链式组合的图像处理管线。

### 第4章：视频帧处理——让画面动起来

从静态图片跨越到动态视频：通过 Canvas 提取视频帧，逐帧送入 Wasm 处理，再实时渲染回画面。理解帧率和流水线的关系。产出：一个能在视频播放时实时施加视觉特效的预览器。

### 第5章：性能优化——跑满每一帧

面对 1080p 实时处理的性能压力，引入 SIMD 并行指令、Web Worker 离线程执行、内存池化等优化手段。理解"接近原生速度"背后实际需要做的工程工作。产出：能够在 1080p 分辨率下维持 30fps 以上实时特效的处理引擎。

### 第6章：视频编码与导出

理解视频编码的基本原理（关键帧、运动估计、码率控制），结合 WebCodecs API 和 Wasm 编码器，将处理后的帧序列编码为可下载的视频文件。产出：完整的视频导出功能。

### 第7章：课程项目整合——完整的 Web 视频编辑器

将前六章的所有模块整合，搭建时间轴编辑器 UI，实现多片段剪辑、转场、导出的完整链路。回顾 Wasm 在整个架构中扮演的角色，复盘性能优化的得失。产出：一个可导入视频、编辑时间轴、施加特效、导出成品的 Web 视频编辑器原型。
