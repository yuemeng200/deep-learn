# 第4章：钩子、事务与软删除

> 前几章我们用 struct 定义了模型、用链式调用完成了查询、用关联表达了关系。本章给这些操作加上**生命周期控制和一致性保障**——让"用 struct 表达关系模型"这件事在真实业务中可靠运行。

## 4.1 钩子——模型的生命周期回调

GORM 在增删改查的特定节点提供了钩子函数，你只需在模型上定义对应方法即可：

```go
// models/task.go
func (t *Task) BeforeCreate(tx *gorm.DB) error {
    if t.Title == "" {
        return errors.New("title is required")
    }
    t.Status = "todo"
    return nil
}

func (t *Task) AfterCreate(tx *gorm.DB) error {
    log.Printf("Task created: ID=%d, Title=%s", t.ID, t.Title)
    return nil
}

func (t *Task) BeforeDelete(tx *gorm.DB) error {
    if t.Status == "in_progress" {
        return errors.New("cannot delete task in progress")
    }
    return nil
}
```

**全部钩子列表**：

| 操作 | 可用钩子 |
|------|---------|
| Create | BeforeCreate, AfterCreate |
| Update | BeforeUpdate, AfterUpdate |
| Delete | BeforeDelete, AfterDelete |
| Find | AfterFind |

**注意事项**：
- 钩子里的 `tx *gorm.DB` 是当前事务，用 `tx` 操作可以和触发钩子的操作在同一事务中
- 返回 `error` 会中止操作并回滚
- `gorm:"-"` 标记的字段不映射数据库，可以在 `AfterFind` 中计算填充

```go
type Task struct {
    gorm.Model
    Title    string
    Priority int
    Urgent   bool  `gorm:"-"`
}

func (t *Task) AfterFind(tx *gorm.DB) error {
    t.Urgent = t.Priority >= 8
    return nil
}
```

## 4.2 事务——保证操作的一致性

**方式一：自动事务（推荐）**

```go
func CreateTaskWithTags(task *Task, tags []Tag) error {
    return DB.Transaction(func(tx *gorm.DB) error {
        if err := tx.Create(task).Error; err != nil {
            return err  // 返回 error 自动回滚
        }
        if err := tx.Model(task).Association("Tags").Append(tags); err != nil {
            return err
        }
        return nil  // 返回 nil 自动提交
    })
}
```

**方式二：手动事务**

```go
func CreateTaskWithTags(task *Task, tags []Tag) error {
    tx := DB.Begin()
    defer func() {
        if r := recover(); r != nil {
            tx.Rollback()
        }
    }()

    if err := tx.Create(task).Error; err != nil {
        tx.Rollback()
        return err
    }
    if err := tx.Model(task).Association("Tags").Append(tags); err != nil {
        tx.Rollback()
        return err
    }
    return tx.Commit().Error
}
```

**嵌套事务（SavePoint）**

```go
DB.Transaction(func(tx *gorm.DB) error {
    tx.Create(&task1)

    tx.Transaction(func(tx2 *gorm.DB) error {
        tx2.Create(&task2)
        return errors.New("rollback task2")  // 只回滚到 SavePoint
    })
    // task1 仍在，task2 被回滚

    return nil
})
```

**实战场景**：任务分配

```go
func AssignTask(taskID, userID uint) error {
    return DB.Transaction(func(tx *gorm.DB) error {
        if err := tx.Model(&Task{}).Where("id = ?", taskID).
            Update("assignee_id", userID).Error; err != nil {
            return err
        }

        log := OperationLog{TaskID: taskID, UserID: userID, Action: "assigned"}
        if err := tx.Create(&log).Error; err != nil {
            return err
        }
        return nil
    })
}
```

## 4.3 软删除

嵌入 `gorm.Model` 自动启用软删除：

```go
DB.Delete(&task, 1)
// UPDATE tasks SET deleted_at = '2026-04-15 10:00:00' WHERE id = 1

DB.Find(&tasks)
// SELECT * FROM tasks WHERE deleted_at IS NULL

DB.Unscoped().Find(&tasks)    // 包含已删除记录
DB.Unscoped().Delete(&task, 1) // 永久删除
```

**唯一索引冲突问题**：软删除的记录仍占唯一索引。解决方案：

```go
type Tag struct {
    gorm.Model
    Name string `gorm:"uniqueIndex:idx_name_not_deleted;where:deleted_at IS NULL"`
}
```

## 4.4 错误处理模式

```go
var task Task
err := DB.First(&task, 999).Error
if errors.Is(err, gorm.ErrRecordNotFound) {
    // 记录不存在
} else if err != nil {
    // 数据库错误
}
```

建议封装 Repository 层：

```go
type TaskRepo struct {
    db *gorm.DB
}

func (r *TaskRepo) GetByID(id uint) (*Task, error) {
    var task Task
    err := r.db.First(&task, id).Error
    if errors.Is(err, gorm.ErrRecordNotFound) {
        return nil, nil
    }
    return &task, err
}

func (r *TaskRepo) Create(task *Task) error {
    return r.db.Create(task).Error
}
```
