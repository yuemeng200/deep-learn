# 第2章：链式查询与预加载

> 上一章我们让 struct 变成了表，完成了单表的增删改查。本章进入 GORM 的核心能力——**用链式调用组合出灵活的查询**，这正是 GORM"可组合性"的具体体现。

## 2.1 链式查询基础

GORM 的查询方法分两类：**链式方法**（返回 `*gorm.DB`，可继续链式调用）和 **终结方法**（执行查询，如 `Find`、`First`）。

```go
// 基础条件
var users []models.User
DB.Where("name = ?", "张三").Find(&users)
DB.Where("name <> ?", "张三").Find(&users)
DB.Where("name IN ?", []string{"张三", "李四"}).Find(&users)
DB.Where("name LIKE ?", "%张%").Find(&users)

// 多条件叠加——链式调用自动用 AND 连接
DB.Where("name = ?", "张三").
    Where("email LIKE ?", "%@test.com").
    Find(&users)
// SELECT * FROM users WHERE name = '张三' AND email LIKE '%@test.com'

// Or 条件
DB.Where("name = ?", "张三").Or("name = ?", "李四").Find(&users)

// Struct 条件（同样受零值限制）
DB.Where(&models.User{Name: "张三"}).Find(&users)
// 只会用 Name 字段，Email 是空字符串会被忽略

// Map 条件（不受零值限制）
DB.Where(map[string]interface{}{"name": "张三", "email": ""}).Find(&users)
```

**Not 条件**

```go
DB.Not("name = ?", "张三").Find(&users)       // WHERE NOT name = '张三'
DB.Not(map[string]interface{}{"name": "张三"}).Find(&users)
```

## 2.2 排序、分页与字段选择

```go
// 排序
DB.Order("created_at desc").Find(&users)

// 分页
var users []models.User
DB.Offset(10).Limit(5).Find(&users)  // 跳过前10条，取5条

// 字段选择
DB.Select("name", "email").Find(&users)       // 只查两个字段
DB.Select("COALESCE(avatar, 'default.png') as avatar").Find(&users)

// 分组与聚合
type Result struct {
    Date  string
    Count int64
}
var results []Result
DB.Model(&models.User{}).
    Select("date(created_at) as date, count(*) as count").
    Group("date(created_at)").
    Having("count > ?", 1).
    Find(&results)
```

## 2.3 Scopes——可复用的查询条件

当你发现某些查询模式反复出现（分页、过滤、软删除检查等），可以用 Scope 抽取：

```go
// 定义 Scope：接收 *gorm.DB，返回 *gorm.DB
func Paginate(page, pageSize int) func(db *gorm.DB) *gorm.DB {
    return func(db *gorm.DB) *gorm.DB {
        offset := (page - 1) * pageSize
        return db.Offset(offset).Limit(pageSize)
    }
}

func NameContains(keyword string) func(db *gorm.DB) *gorm.DB {
    return func(db *gorm.DB) *gorm.DB {
        return db.Where("name LIKE ?", "%"+keyword+"%")
    }
}

// 使用
DB.Scopes(NameContains("张"), Paginate(1, 10)).Find(&users)
// SELECT * FROM users WHERE name LIKE '%张%' LIMIT 10 OFFSET 0
```

Scopes 的价值在于**组合**——多个 Scope 可以像搭积木一样拼装查询，比到处复制 Where 条件干净得多。

## 2.4 预加载——解决 N+1 问题

当模型有关联关系时（下一章会详细讲），直接查列表会导致 N+1 问题：

```go
// 假设 Task 有一个 Creator 字段关联到 User
var tasks []models.Task
DB.Find(&tasks)
for _, t := range tasks {
    // 每条 task 都会触发一次查询 user！100 条 task = 100 次额外查询
    fmt.Println(t.Creator.Name)
}
```

**Preloading 提前加载关联数据**，一次搞定：

```go
// 基础预加载
DB.Preload("Creator").Find(&tasks)
// 执行两条 SQL：
// 1. SELECT * FROM tasks
// 2. SELECT * FROM users WHERE id IN (1, 2, 3, ...)

// 嵌套预加载
DB.Preload("Creator").Preload("Comments").Find(&tasks)

// 带条件的预加载
DB.Preload("Comments", "status = ?", "approved").Find(&tasks)
// 只加载已审核的评论
```

**Preload vs Joins**

```go
// Preload：两条 SQL，用 IN 关联（推荐，默认方式）
DB.Preload("Creator").Find(&tasks)

// Joins：一条 SQL，用 JOIN（适合需要根据关联字段筛选的场景）
DB.Joins("Creator").Where("Creator.name = ?", "张三").Find(&tasks)
```

简单记忆：**只查数据用 Preload，需要按关联字段过滤用 Joins**。

## 2.5 子查询

```go
// 子查询作为条件
subQuery := DB.Model(&models.User{}).Select("id").Where("name LIKE ?", "%张%")
DB.Where("creator_id IN (?)", subQuery).Find(&tasks)

// From 子查询
DB.Table("(?) as u", DB.Model(&models.User{}).Select("name", "email")).
    Find(&results)
```
