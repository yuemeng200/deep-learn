# 在 Go 中使用 Redis — 渐进式学习课程

## 核心本质

Redis 的根本命题是：**用内存换时间**。当磁盘数据库成为瓶颈，Redis 提供了一个极简的内存数据结构服务器——它不试图做万能数据库，而是用"有限但精准的数据结构 + 单线程无锁模型"换来了极致的速度和可预测的延迟。

理解 Redis 的关键不在于记住多少命令，而在于**学会用正确的数据结构匹配正确的场景**。每种数据结构都是对一类真实问题的直接抽象。

## 实战项目

**「简易内容社区后端」** —— 一个支持用户发帖、热门排行、内容缓存、访问限流的轻量社区 API。

选择它的理由：这个项目能自然引出 Redis 的五大数据结构（String、Hash、List、Set、Sorted Set）、缓存设计、会话管理、限流和发布订阅，覆盖面广且每章都有可运行的增量产出。

## 课程大纲

**第1章：初识 Redis 与 Go 客户端**
理解 Redis 是什么、为什么快，搭建 Go + Redis 开发环境，使用 go-redis 完成第一次读写。产出：能跑通的项目骨架 + 健康检查接口。

**第2章：五大数据结构与场景映射**
系统学习 String、Hash、List、Set、Sorted Set，重点不是命令罗列，而是"什么场景该用什么结构"。产出：用户资料存储（Hash）、发帖与时间线（List）、标签系统（Set）、热门排行榜（Sorted Set）。

**第3章：缓存设计 — 从能用到好用**
掌握缓存读写模式（Cache-Aside）、TTL 策略，理解缓存穿透、雪崩、击穿三大经典问题及应对方案。产出：为帖子详情接口加上缓存层，性能显著提升。

**第4章：会话管理与访问限流**
用 Redis 实现用户会话存储和 API 限流（滑动窗口），理解原子操作在并发场景中的价值。产出：登录态管理 + 接口限流中间件。

**第5章：发布订阅与消息队列**
学习 Pub/Sub 实现实时通知，用 List 构建简易消息队列，理解 Redis 在异步场景中的角色与局限。产出：新帖实时推送 + 异步通知处理。

**第6章：项目整合与生产实践**
连接池调优、Pipeline 批量操作、事务与 Lua 脚本、错误处理与重试策略。将前五章的功能整合为完整可部署的服务，并做复盘。产出：完整的社区后端项目。
